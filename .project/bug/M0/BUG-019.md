## Bug 描述

Settings 页面修改颜色模式（深色/浅色），只影响 Settings 页面本身。Tasks/Bugs/Logs 等其他页面始终保持浅色，不受 Settings 主题设置影响。

## 根因分析

**Settings 页面**：`updateTheme()` 正确保存主题到 `localStorage.theme` 和 `/api/settings`，并设置当前页面的 `data-theme` 属性 ✅

**Tasks/Bugs/Logs 页面**：这三个页面都有自己独立的 inline JavaScript，有自己的 `DOMContentLoaded` 初始化函数，**完全不调用 `initTheme()` 或 `loadSettings()`**，也从 `localStorage` 读取主题。

导致：用户从 Settings 切换深色模式后，Settings 页面显示深色 ✅，但导航到 Tasks/Bugs/Logs 时，这些页面以空白 `data-theme` 属性加载，`theme.css` 的 `:root` 默认值（意外地是深色值），与页面实际内容颜色不匹配。

**额外发现**：`theme.css` 的 `:root` CSS 变量包含**深色值**（`--sidebar-bg: #1E293B`），而非浅色值。这意味着：
- 无 `data-theme` 属性时：侧边栏深色，页面内容浅色 → **颜色严重不匹配**
- `data-theme="dark"` 时：侧边栏更深的深色，页面内容也变深 → 看起来一致

## 期望行为

深色模式下整个应用（所有页面）都应呈现深色；浅色模式下都应呈现浅色。

## 实际行为

Settings 页面主题切换正常。其他页面（Tasks/Bugs/Logs）不受 Settings 主题设置影响。

## 修复方案

### 1. Tasks/Bugs/Logs 页面主题初始化
在每个页面的 `DOMContentLoaded` 开始处添加：

```javascript
// 读取并应用保存的主题
const savedTheme = localStorage.getItem('theme');
if (savedTheme) {
  document.documentElement.setAttribute('data-theme', savedTheme);
}
```

### 2. theme.css :root 修复
`theme.css` 的 `:root` 应包含**浅色值**，`[data-theme="dark"]` 覆盖为深色值。当前 `:root` 里的 `--sidebar-*` 是深色值，应与 `[data-theme="dark"]` 块中的值对调。
