## Bug 描述

Bugs 页面的 Bug 趋势图表（Bug Trend Chart）完全不显示数据，空白。

## 验证结果

API `/api/bugs/trend?days=7` 返回近 7 天数据，全部为 0：

```
date: 2026-03-18 ~ 2026-03-24
total: 0, fixed: 0, open: 0  (每日)
```

项目创建于 2026-03-19，BUG 跟踪从 2026-03-21 开始，但 `/api/bugs/trend` 的每日统计始终为 0，无历史累计数据。

## 根因分析

`/api/bugs/trend` 数据来源于历史记录，但系统从未每日记录 bug 总数/修复数，仅在查询时实时计算当日快照，导致趋势数据全为 0。

## 修复方案

方案 A（数据修复）：`/api/bugs/trend` 应维护每日 bug 统计数据（total/open/fixed），在 bug 状态变更时更新历史累计。

方案 B（前端容错）：如果 API 无历史趋势数据，图表应优雅提示"暂无趋势数据"，而非显示空白。

## 影响评估

Minor：趋势图无数据不影响核心功能，不阻塞发布。建议非紧急修复。
