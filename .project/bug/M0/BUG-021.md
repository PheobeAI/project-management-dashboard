## Bug 描述

整个管理系统网页布局全部乱了，UI挤在一起。

## 初步分析

与 BUG-019 的 `theme.css` `:root` CSS 变量问题相关：

`theme.css` 的 `:root` 中 `--sidebar-*` 变量被设为深色值（`#1E293B` 等），而非浅色值。导致：

- 无 `data-theme` 属性（初始状态）：侧边栏深色背景 + 白色文字，主体内容区浅色背景 → **颜色严重不匹配，视觉上像"布局挤在一起"**
- `data-theme="dark"` 时：整体深色，视觉正常

同时，`/assets/css/variables.css` 在 `layout.hbs` 中被引用，但该文件在 `src/assets/css/` 下**不存在**（src/public/css/ 下也无此文件），导致 404。该文件可能是布局 CSS 变量的来源，但已缺失。

## 期望行为

浅色模式下：侧边栏浅色背景 + 深色文字，内容区协调浅色背景。

## 实际行为

`theme.css` :root 意外包含深色值，浅色模式初始状态侧边栏即呈现深色，与内容区颜色不匹配。

## 修复方案

1. 将 `theme.css` `:root` 中的 `--sidebar-*` 值替换为浅色值（参考 main.css 中 .sidebar 选择器内的注释 "Light Mode" 值）
2. 确认 `/assets/css/variables.css` 是否为必需的 CSS 变量文件，如需要则创建，或从 layout.hbs 中移除该引用
