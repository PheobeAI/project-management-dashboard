## Bug 描述

打开 Settings 页面和控制台报错：
```
app.js:1 Uncaught SyntaxError: Identifier 'API_BASE' has already been declared
```

Projects 页面 themeBtn onclick 报错：
```
projects:91 Uncaught ReferenceError: toggleTheme is not defined
```

## 根因分析

**同一根因的两个表现：**

`layout.hbs` 加载 `<script src="/static/js/app.js">`，app.js 内有 `const API_BASE = '/api'`。

projects.hbs / agents.hbs / bugs.hbs / tasks.hbs / settings.hbs / documents.hbs / versions.hbs 这些页面模板的**内联 `<script>` 块**里也有 `const API_BASE = '/api'`。

当 layout.hbs 与页面内容同时渲染时，app.js 的 `const API_BASE` 和页面内联脚本的 `const API_BASE` 在**同一全局作用域**，导致重复声明错误：

```
app.js: const API_BASE = '/api'  ← OK
projects.hbs inline: const API_BASE = '/api'  ← SyntaxError: already declared
```

**为什么 toggleTheme is not defined**：
因为 `const API_BASE` 报 SyntaxError 后，该行**之后的 JavaScript 均不执行**。所以 app.js 中 `toggleTheme` 的定义也失效了（如果 `const API_BASE` 在 app.js 的前几行）。

## 受影响页面

- projects.hbs（`const API_BASE` 在 line 42）
- agents.hbs（`const API_BASE` 在 line 39）
- bugs.hbs（`const API_BASE` 在 line 93）
- tasks.hbs（`const API_BASE` 在 line 86）
- settings.hbs（`const API_BASE` 在 line 189）
- documents.hbs（`const API_BASE` 在 line 63）
- versions.hbs（`const API_BASE` 在 line 31）

## 期望行为

所有页面正常加载，无 JS 语法错误。

## 实际行为

页面 JS 因 API_BASE 重复声明报 SyntaxError，后续所有 JS（包含 toggleTheme）失效。

## 修复方案

从所有 7 个页面模板的内联 `<script>` 块中**删除** `const API_BASE = '/api';` 这一行（app.js 已提供全局 `API_BASE`）。

受影响文件：
- src/views/projects.hbs（line 42）
- src/views/agents.hbs（line 39）
- src/views/bugs.hbs（line 93）
- src/views/tasks.hbs（line 86）
- src/views/settings.hbs（line 189）
- src/views/documents.hbs（line 63）
- src/views/versions.hbs（line 31）
