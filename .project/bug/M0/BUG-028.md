# BUG-028: 多个 .hbs 文件 inline script 语法错误导致页面无法渲染

## Severity
critical

## Status
verified

## 发现的版本
M0

## Environment
- OS: Windows 10 x64
- Browser: Edge (已登录态)
- 测试时间: 2026-03-31 13:56 GMT+8
- App版本: project-management-dashboard (latest)

## 受影响的页面
| 页面 | 文件 | 当前状态 |
|------|------|---------|
| /agents | agents.hbs | loading spinner |
| /projects | projects.hbs | loading spinner |
| /versions | versions.hbs | loading spinner |
| /documents | documents.hbs | loading spinner |
| /bugs | bugs.hbs | loading spinner |

**注意**：/dashboard 正常（home.hbs 无语法错误）

## Steps to Reproduce
1. 启动服务器: `node src/server/index.js`
2. 打开浏览器访问 http://localhost:3000/agents
3. 观察页面状态

## Expected
各页面应正常加载并显示数据

## Actual
页面一直显示 loading spinner，数据无法渲染。控制台有 JS 语法错误。

## Root Cause
多个 .hbs 文件的 inline `<script>` 中存在 **template literal 语法错误**，导致整个 `<script>` 块解析失败，函数定义全部失效。

**诊断方法**：
```javascript
// Node.js 中对每个 .hbs 文件执行
const m = content.match(/<script>([\s\S]*?)<\/script>/);
new Function(m[1]);  // 抛出 SyntaxError
```

**具体错误**：

| 文件 | 错误描述 |
|------|---------|
| agents.hbs Line 87 | `id="toggle-${agent.name}">` — `${agent.name}` 缺少闭合 `}` |
| versions.hbs Line 84 | `${changes.length > 0 ? \`` — template literal 从句未闭合 |
| projects.hbs | Invalid or unexpected token |
| tasks.hbs | Unexpected token ':' |
| documents.hbs | Invalid or unexpected token |
| bugs.hbs | Unexpected token 'else' |

所有错误都是 **Handlebars template literal 内 `${...}` 缺少闭合 `}`** 或 **反引号未正确配对**。

## Evidence
```
agents.hbs: SYNTAX ERROR - Invalid or unexpected token
  Line 87 has mismatched ${}: open=1 close=0
  Content: <div class="agent-toggle" id="toggle-${agent.name}">

versions.hbs: SYNTAX ERROR - Invalid or unexpected token
  Line 84 has mismatched ${}: open=1 close=0
  Content: ${changes.length > 0 ? `
```

## Fix
修复这些 .hbs 文件中 template literal 的语法错误：
1. agents.hbs: `${agent.name}` → 找到对应闭合 `</div>` 前的完整 template literal 修复
2. versions.hbs: 修复 `${changes.length > 0 ? \`...\` : ''}` 的闭合
3. 其他文件：逐一检查并修复

**注意**：Handlebars `{{#each}}` 和 `${}` JS 表达式混用时，注意区分 `}}`（Handlebars）和 `}`（JS）闭合。

## Test Script
修复后刷新页面，所有受影响页面应：
1. 不再显示 loading spinner
2. 正常显示数据内容
3. 浏览器控制台无 SyntaxError
