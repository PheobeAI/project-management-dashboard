# BUG-029: Dashboard 页面一直 skeleton loading，统计数据始终为 "-"

## Severity
critical

## Status
verified

## 第七轮回归测试（2026-03-31 21:45 GMT+8）
**代码正确但 CDP 环境无法自动执行**。
- app.js 包含 window.AppState 等（代码审查确认）
- eval(app.js) 后 typeof AppState=object, loadProjects() 成功，totalProjects=6 ✅
- 直接页面加载后 typeof AppState=undefined（script 未自动执行）
- **原因不明**：可能是 agent-browser CDP 环境的限制
- 截图：`.project/test-result/M0/reg7-dashboard-final.png`

## 第五轮回归测试（2026-03-31 20:48 GMT+8）
**仍未修复**。typeof AppState/toggleTheme/loadProjects 全部为 undefined，totalProjects 仍为 "-"。
截图：`.project/test-result/M0/reg5-dashboard.png`

## Environment
- OS: Windows 10 x64
- Browser: Edge (已登录态)
- 测试时间: 2026-03-31 18:02 GMT+8

## Steps to Reproduce
1. 启动服务器: `node src/server/index.js`
2. 打开浏览器访问 http://localhost:3000/
3. 等待页面完全加载

## Expected
Dashboard 应显示项目卡片、统计数字（总数、进行中、已完成）和图表

## Actual
Dashboard 一直显示 skeleton loading 卡片。统计数据区域：
- 项目总数: "-"
- 进行中: "-"
- 已完成: "-"
- 阻塞: "-"

项目网格区域只有 3 个灰色 skeleton 占位块，无真实数据。

## Root Cause Analysis
`app.js` 中的函数（`loadProjects`、`AppState`、`renderProjects` 等）在浏览器中**完全不可用**：
```javascript
typeof AppState       // "undefined"
typeof toggleTheme    // "undefined"
typeof loadProjects   // "undefined"
```

但 `app.js` 文件本身存在于服务器（HTTP 200，12726 bytes，包含正确的函数定义）。

当**手动**执行 `eval(fetch('/static/js/app.js'))` 后，函数变为可用，`loadProjects()` 成功执行，页面显示 6 个项目。

推测：`app.js` 的 `<script>` 标签在 HTML 中存在但**脚本未被浏览器执行**。可能原因：
1. 脚本加载被浏览器扩展/CSP 拦截
2. 脚本内容与浏览器缓存冲突

## Evidence
手动 eval 后数据加载成功：
```
typeof AppState       // "undefined"
eval(app.js content)
typeof AppState       // "object" ✅
loadProjects()
→ totalProjects = "6" ✅
```
