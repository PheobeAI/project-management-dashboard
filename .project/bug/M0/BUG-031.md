major

## Status
verified

## 第五轮回归测试（2026-03-31 20:48 GMT+8）
**仍未修复**。侧边栏导航项仍聚集在顶部。截图：`.project/test-result/M0/reg5-dashboard.png`

## Environment
- OS: Windows 10 x64
- Browser: Edge (已登录态)
- 测试时间: 2026-03-31 18:02 GMT+8

## Steps to Reproduce
1. 启动服务器: `node src/server/index.js`
2. 打开浏览器访问任意页面
3. 观察左侧侧边栏布局

## Expected
侧边栏导航选项应均匀分布，占据整个侧边栏高度

## Actual
所有导航选项（Dashboard、Projects、Agents、Tasks、Versions、Documents、Bugs、Log、Settings）**全部聚集在侧边栏顶部**，下方留下大量空白区域，只有最底部的"收缩侧边栏"按钮。

## Evidence
截图可见：`.project/test-result/M0/boss-bug1-dashboard.png`

侧边栏截图显示约 9 个导航项全部挤在上半部分，下方超过 50% 的区域为空。

## Possible Cause
CSS 布局问题：
- 侧边栏可能使用了固定高度或 flexbox 分布不均
- 导航项的 `flex` 属性可能设置不当
- 侧边栏底部按钮可能被 `position: absolute` 固定而非正常流布局

## Suggested Investigation
检查 `sidebar.css` 中的以下属性：
- `align-items` / `justify-content`
- `flex-direction` / `flex-grow`
- 侧边栏高度设置
