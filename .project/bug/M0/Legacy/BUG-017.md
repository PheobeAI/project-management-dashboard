## Bug 描述

右上角主题切换按钮点击后无法切换深色模式（侧边栏和主内容区都不生效）。

## 期望行为

在 Settings 选择深色模式后，跳转到 Tasks/Projects 等其他页面时主题保持深色。

## 实际行为

1. 在 Settings 将主题切换为"深色模式"，下拉框正确显示已选中
2. 跳转到 Tasks 页面，`data-theme` 属性为 `null`，页面显示为浅色

---

## 根因分析

**双重机制导致主题无法跨页面持久化：**

### 机制 1：Settings 页面（写入 → API）
Settings 页面的 `updateTheme()` 将主题保存到 **API**：
```javascript
// settings.hbs inline script
async function updateTheme() {
  const theme = document.getElementById('themeSelect').value;
  document.documentElement.setAttribute('data-theme', theme);
  await saveSettings({ theme }); // 只写入 API，不写 localStorage！
}
```

### 机制 2：其他页面（读取 ← localStorage）
Dashboard/Tasks/Projects 等页面通过 `app.js` 的 `initTheme()` 从 **localStorage** 读取：
```javascript
// app.js
function initTheme() {
  const savedTheme = localStorage.getItem('theme') || 'light'; // 只读 localStorage！
  document.documentElement.setAttribute('data-theme', savedTheme);
}
```

**两个机制完全隔离**：Settings 写 API，其他页面读 localStorage，永远对不上。

### Engineer 第一轮修复（settings.hbs loadSettings）
添加了 `document.documentElement.setAttribute('data-theme', settings.theme)` 到 `loadSettings()`，确保 Settings 页面加载时正确应用主题。

**测试结果：✅ 有效** — Settings 页面自身的主题加载问题已解决。

### 遗留问题：跨页面不持久化
即使 Settings 正确写入了 API，其他页面（Dashboard/Tasks/Projects）也不从 API 读取主题，因为它们只读 `localStorage`。

## 完整修复方案

**在 Settings 页面的 `updateTheme()` 中，同时写入 localStorage 和 API：**

```javascript
async function updateTheme() {
  const theme = document.getElementById('themeSelect').value;
  
  // 应用到 DOM
  if (theme === 'system') {
    const prefersDark = window.matchMedia('(prefers-color-scheme: dark)').matches;
    document.documentElement.setAttribute('data-theme', prefersDark ? 'dark' : 'light');
  } else {
    document.documentElement.setAttribute('data-theme', theme);
  }
  
  // ✅ 同时保存到 localStorage（供其他页面读取）
  localStorage.setItem('theme', theme);
  
  // 保存到 API
  await saveSettings({ theme });
}
```

或者：统一所有页面从 API 读取主题（需要修改 `app.js` 的 `initTheme()`）。
