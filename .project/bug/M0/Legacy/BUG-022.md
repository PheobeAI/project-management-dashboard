## Bug 描述

**两个表现（同一根因）：**
1. **布局乱**：浅色模式下侧边栏为深色 #1E293B，内容区为浅色，严重颜色不匹配
2. **主题不传递**：Settings 修改主题后，Dashboard/Projects/Agents/Versions/Documents 等页面主题不跟随变化（侧边栏始终深色）

## 根因分析

**CSS 变量覆盖链（加载顺序）：**

1. `main.css`（line 25）：`:root { --sidebar-bg: #F1F5F9; }` — 浅色 ✅
2. `theme.css`：`:root { --sidebar-bg: #1E293B; }` — **深色** ❌
3. `variables.css`：`:root { --sidebar-bg: #1E293B; }` — **深色** ❌（来自 --color-sidebar，非 --sidebar-bg，但最终结果相同）

`sidebar.css` 使用 `var(--sidebar-bg)`，该变量最终被 `variables.css :root` 的 `#1E293B` 覆盖，导致：

- 无 `data-theme` 属性（初始/浅色模式）：侧边栏深色 #1E293B，页面内容区浅色 #F8FAFC — **严重颜色不匹配**
- `data-theme="dark"` 时：侧边栏 #0F172A，内容区也深色 — 看起来一致

**加载顺序**（layout.hbs）：
```html
<link rel="stylesheet" href="/static/css/main.css">       <!-- line 8 -->
<link rel="stylesheet" href="/static/css/theme.css">     <!-- line 9 -->
<link rel="stylesheet" href="/static/css/sidebar.css">    <!-- line 12 -->
<link rel="stylesheet" href="/assets/css/variables.css">  <!-- line 13 -->
```

## 期望行为

浅色模式下：侧边栏浅色背景 + 深色文字，内容区协调浅色背景。

## 实际行为

浅色模式下侧边栏始终为深色 #1E293B，与浅色内容区严重不匹配。

## 修复方案

### 方案（推荐）：调整 CSS 加载顺序
将 `variables.css` 移到 `main.css` **之前**加载，使其 `:root --sidebar-bg: #1E293B` 被 `main.css :root --sidebar-bg: #F1F5F9` 覆盖：
```html
<link rel="stylesheet" href="/assets/css/variables.css">  <!-- 先加载，深色 -->
<link rel="stylesheet" href="/static/css/main.css">       <!-- 后加载，浅色覆盖 -->
<link rel="stylesheet" href="/static/css/theme.css">     <!-- [data-theme=dark] 覆盖块保留 -->
```

### 备选方案：从 variables.css 中移除 --sidebar-bg 定义
variables.css 的 `:root` 中 `--sidebar-bg: #1E293B` 是深色值，与 variables.css 作为"浅色规范"的角色矛盾。应从 variables.css 中删除 `--sidebar-bg` 定义，仅保留 `--color-sidebar`（如果该变量被某处使用）。
