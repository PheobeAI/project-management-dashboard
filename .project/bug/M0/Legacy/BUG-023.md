## Bug 描述

在 Settings 页面，点击「添加路径」后点击「取消」，之后 Settings 页面所有路径操作（编辑、添加新路径）均失效，无法编辑。

## 根因分析

**showAddPathModal() 的 confirmBtn.onclick 未重置**

`hideAddPathModal()` 关闭 Modal 时：
- 重置表单字段（newPathId, newPathInput, newPathAlias, newPathColor）
- 重置 Modal 标题（通过 hideAddPathModal）
- **但 confirmBtn.onclick 仍保持为 editPath() 中设置的 updatePath**

editPath() 修改 Modal 状态：
```js
confirmBtn.onclick = updatePath;  // editPath 中设置
title.textContent = '编辑路径';
```

showAddPathModal() 重新打开时：
```js
// 未重置 confirmBtn.onclick！
modal.style.display = 'flex';  // 显示 Modal，但 confirmBtn 仍指向 updatePath
```

**结果**：点击"添加路径"后，confirmBtn 仍执行 updatePath 而非 addPath，导致操作失败。

## 复现步骤

1. 打开 Settings 页面
2. 点击「添加路径」按钮 → Modal 打开
3. 点击「取消」→ Modal 关闭
4. 再次点击「添加路径」按钮 → Modal 打开
5. 此时 confirmBtn 仍指向 updatePath，点击确认添加时执行错误函数

## 期望行为

每次打开添加路径 Modal 时，confirmBtn 应执行 addPath；编辑 Modal 时执行 updatePath。

## 实际行为

editPath → Cancel → 再次 showAddPathModal 时，confirmBtn 仍为 updatePath，导致添加路径操作失败。

## 修复方案

在 `showAddPathModal()` 开头重置 confirmBtn 状态：

```js
function showAddPathModal() {
  document.getElementById('newPathId').value = '';
  document.getElementById('newPathInput').value = '';
  document.getElementById('newPathAlias').value = '';
  document.getElementById('newPathColor').value = '#3B82F6';
  document.getElementById('pathModalTitle').textContent = '添加项目路径';
  
  // 重置 confirmBtn 状态
  const confirmBtn = document.querySelector('#addPathModal .btn-primary');
  if (confirmBtn) {
    confirmBtn.textContent = '添加';
    confirmBtn.onclick = addPath;
  }
  
  const modal = document.getElementById('addPathModal');
  modal.style.display = 'flex';
  // ...
}
```
