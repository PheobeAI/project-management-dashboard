# BUG-026: Dashboard skeleton loading 一直显示

## Severity
critical

## Status
open

## 发现的版本
M0

## Environment
- OS: Windows 10 x64
- Browser: Edge (已登录态)
- 测试时间: 2026-03-31 13:30 GMT+8
- App版本: project-management-dashboard (latest)

## Steps to Reproduce
1. 启动服务器: `node src/server/index.js`
2. 打开浏览器访问 http://localhost:3000/
3. 等待页面完全加载

## Expected
Dashboard 应显示项目卡片、统计数字和图表

## Actual
Dashboard 一直显示 skeleton loading 卡片，无法渲染真实数据。

## Root Cause
inline scripts 中的 `DOMContentLoaded` 监听器在 HTML 解析阶段注册，但 `API_BASE` 在 app.js 中以 `const` 声明，不在 `window` 上，导致 `${API_BASE}` 抛出 `ReferenceError`。该错误被 `window.onerror` 捕获后阻断了 `app.js` 自身 DOMContentLoaded 中的 `loadProjects()` 执行。

**第一次修复（defer → head）无效的原因**：`defer` 脚本在 HTML 解析完成后才执行，晚于 inline scripts 的 `DOMContentLoaded` 注册时机。

## Fix
**Files modified:**
- `src/views/layout.hbs` — 在 `<head>` 中添加内联脚本 `<script>window.API_BASE = '/api';</script>`，在内联脚本执行前就已设置
- `src/public/js/app.js` — 添加 `window.API_BASE = '/api'`（保留作为 fallback）

**方案说明**：内联 `<script>` 在 HTML 解析阶段同步执行，早于 `{{{content}}}` 中的任何 inline script，也早于 `DOMContentLoaded` 事件触发，确保 `window.API_BASE` 在所有 `DOMContentLoaded` 监听器执行时已存在。
