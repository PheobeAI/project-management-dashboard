# BUG-027: 6个页面因 API_BASE 未定义导致数据加载失败

## Severity
critical

## Status
verified

## 影响的页面
- `/agents` — Agent 工作负载页面
- `/projects` — 项目列表页面
- `/tasks` — 任务列表页面
- `/versions` — 版本历史页面
- `/documents` — 文档页面
- `/bugs` — Bug 追踪页面

## Environment
- OS: Windows 10 x64
- Browser: Edge (已登录态)
- 测试时间: 2026-03-31 13:30 GMT+8

## Steps to Reproduce
1. 启动服务器: `node src/server/index.js`
2. 打开浏览器访问 http://localhost:3000/agents
3. 观察页面状态

## Expected
各页面应正常加载并显示数据

## Actual
所有受影响的页面都卡在 loading spinner 或 skeleton 状态，控制台报 `ReferenceError: API_BASE is not defined`。

## Root Cause
`layout.hbs` 的 HTML 结构：
```html
<head>...</head>
<body>
  ...
  <main class="main-content">
    {{{content}}}   ← agents.hbs 等页面的内容，含 inline <script>
  </main>
  ...
  <script src="/static/js/app.js"></script>  ← 在 body 末尾
</body>
```

各页面的 inline `<script>` 中：
```javascript
document.addEventListener('DOMContentLoaded', () => {
  loadAgents();  // DOMContentLoaded 触发时，API_BASE 尚不存在
});
async function loadAgents() {
  const response = await fetch(`${API_BASE}/agents`);  // ReferenceError!
}
```

`const API_BASE` 在 app.js 中，不在 `window` 上，inline scripts 执行时 `API_BASE` 不在作用域内。

**第一次修复（defer → head）无效的原因**：`defer` 脚本在 HTML 解析完成后才执行，DOMContentLoaded 在此之前已触发，inline scripts 的监听器已经注册但此时 API_BASE 仍不存在。

## Fix
**Files modified:**
- `src/views/layout.hbs` — 在 `<head>` 中添加内联脚本 `<script>window.API_BASE = '/api';</script>`
- `src/public/js/app.js` — 添加 `window.API_BASE = '/api'`（保留作为 fallback）

**方案说明**：内联 `<script>` 在 HTML 解析阶段同步执行，早于 `{{{content}}}` 中的任何 inline script 和 DOMContentLoaded 事件，确保 `window.API_BASE` 在所有使用它的代码执行前就已存在。

## Evidence
```javascript
// 在受影响页面执行
typeof API_BASE          // 应返回 'string'（修复后）
fetch('/api/agents')     // 应返回 {success: true, data: [...]}
```
