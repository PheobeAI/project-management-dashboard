# 03 - 非功能需求

## 3.1 性能要求

| 指标 | 要求 |
|:-----|:-----|
| 首屏加载时间 | ≤ 2 秒（数据加载完成可交互） |
| 数据刷新响应 | ≤ 1 秒 |
| 页面切换响应 | ≤ 300ms |
| 动画帧率 | ≥ 30 FPS，理想 60 FPS |
| 多路径扫描 | 支持 10+ 路径，每个路径下 50+ 项目的扫描性能 |

## 3.2 兼容性要求

| 平台 | 最低版本 |
|:-----|:---------|
| Chrome | 90+ |
| Edge | 90+ |
| Firefox | 88+ |
| Safari | 14+ |
| Windows | 10+ |

## 3.3 可用性要求

- **响应式设计**：支持 1024px 以上桌面屏幕
- **键盘导航**：支持 Tab 键切换焦点
- **错误提示**：数据加载失败时显示友好提示，不白屏
- **加载状态**：数据加载中显示骨架屏或加载动画
- **主题切换**：深色/浅色模式切换流畅，无闪烁

## 3.4 视觉与动画要求

- **动画效果**：页面切换、卡片悬停、数据更新、主题切换等场景应有流畅动画
- **过渡时间**：CSS transitions 推荐 200-300ms
- **动效曲线**：使用 ease-out 或 cubic-bezier 曲线，避免线性动画
- **深色模式**：保证对比度，确保文字和元素在深色背景下清晰可见

## 3.5 扩展性要求

- 数据源目录结构可通过配置文件调整
- 支持新增图表类型
- 前端框架模块化，便于维护
- 支持插件化扩展新功能

## 3.6 技术约束

- 禁止使用 IE 11
- 不支持移动端（仅桌面端）
- 后端服务与前端同机运行，无需网络请求优化

## 3.7 版本管理要求

### 3.7.1 GitHub 集成

| 需求项 | 要求 |
|:-------|:-----|
| 仓库类型 | Private GitHub 仓库 |
| 仓库命名 | `project-management-dashboard` 或类似 |
| 提交范围 | 整个项目目录（包括 `.project/` 目录） |
| 提交频率 | 每次状态变更、任务更新后自动提交 |
| 提交信息 | 包含变更类型和简要描述 |

### 3.7.2 版本控制内容

| 目录/文件 | 版本控制 | 说明 |
|:----------|:---------|:-----|
| `.project/status.json` | ✅ 必须 | 项目核心状态 |
| `.project/tasks/*.md` | ✅ 必须 | 所有任务文件 |
| `.project/reviews/` | ✅ 必须 | 评审记录 |
| `.project/logs/` | ✅ 必须 | 操作日志 |
| 源代码目录 | ✅ 必须 | 开发代码 |
| 测试数据 | ✅ 建议 | 测试用例和结果 |

### 3.7.3 GitHub 集成功能

- **自动提交**：状态变更时自动推送到 GitHub
- **分支策略**：使用 `main` 分支作为主分支
- **.gitignore**：排除构建产物（`build/`、`dist/`、`node_modules/` 等）
- **GitHub Token**：用户配置个人 Access Token 用于认证
