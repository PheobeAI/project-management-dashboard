# 04 - 用户故事

## 4.1 用户故事列表

| 编号 | 角色 | 故事描述 | 验收条件 |
|:-----|:-----|:---------|:---------|
| US1 | Boss | 作为 Boss，我想在首页一眼看到所有项目的状态概览，以便快速识别哪些项目正常、哪些需要关注 | 项目列表卡片展示项目名称、进度、状态指示器 |
| US2 | Boss | 作为 Boss，我想看到各项目的进度对比图表，以便了解项目间的相对进展 | 展示项目完成度对比柱状图或饼图 |
| US3 | PM | 作为 PM，我想点击项目卡片查看详细信息，以便了解具体任务和测试情况 | 点击卡片可进入项目详情页 |
| US4 | PM | 作为 PM，我想看到每个项目的任务列表和完成率，以便跟踪任务进度 | 详情页展示任务列表和完成率统计 |
| US5 | PM | 作为 PM，我想看到测试结果汇总，以便了解产品质量状态 | 展示测试通过率、失败测试列表 |
| US6 | PM | 作为 PM，我想知道当前每个项目在等待哪个 Agent 处理，以便协调工作 | 展示 waiting_for 信息 |
| US7 | Engineer | 作为 Engineer，我想看到项目的测试状态，以便了解需要修复的问题 | 展示测试结果和失败用例 |
| US8 | 所有用户 | 作为平台用户，我希望数据能够自动刷新，以便随时看到最新状态 | 支持自动刷新和手动刷新 |
| US9 | 所有用户 | 作为平台用户，我希望切换深色/浅色模式，以便在不同环境下舒适使用 | 主题切换按钮存在，切换后所有页面元素主题同步变化 |
| US10 | PM | 作为 PM，我想添加多个项目路径，以便在统一界面管理不同位置的项目 | 支持添加、编辑、删除路径，路径下的项目正常显示 |
| US11 | PM | 作为 PM，我想为不同路径设置别名和颜色标签，以便快速识别项目来源 | 路径可设置别名和颜色，项目卡片显示路径来源 |
| US12 | 所有用户 | 作为平台用户，我希望查看各 Agent 的任务列表和状态，以便了解团队工作负载 | Agent 任务视图展示各 Agent 的 waiting/in_progress/done 任务 |
| US13 | PM | 作为 PM，我想看到项目当前处于哪个阶段（Phase 0-8），以便了解项目进展 | 阶段进展视图展示当前 Phase 和阶段时间线 |
| US14 | QA | 作为 QA，我想看到 Bug 统计和修复进度，以便跟踪质量问题 | 测试详情视图展示 Bug 数量、修复率、评审意见 |
| US15 | PM | 作为 PM，我想添加项目路径时调起目录选择对话框，以便快速选择文件夹 | 添加路径按钮调起系统目录选择对话框 |
| US16 | 所有用户 | 作为平台用户，我希望加载失败时显示内联提示，而非弹窗打断工作 | 错误使用内联提示，不显示阻断式弹窗 |
| US17 | 所有用户 | 作为平台用户，我希望筛选任务时正确显示结果，不再出现空列表 | 筛选逻辑正确，筛选结果正确显示 |

## 4.2 优先级排序

1. **US1, US2, US3** — 核心价值，必须实现
2. **US4, US5, US6** — 重要功能，提升用户体验
3. **US7, US8** — 增强功能，提升可用性
4. **US9, US10, US11** — v2.0 新增功能，深色模式和多路径管理
5. **US12, US13, US14, US15, US16, US17** — v3.0 修复功能，Agent 视图、阶段进展、Bug 视图、UI 修复
