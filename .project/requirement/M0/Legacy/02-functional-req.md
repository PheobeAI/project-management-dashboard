## 2.2 详细功能说明

### F1 项目列表与概览

**描述**：在首页展示所有已配置项目路径下项目的卡片式列表。

**功能要求**：
- 自动扫描所有已配置路径下的子目录（排除隐藏目录和系统目录）
- 每个项目卡片显示：
  - 项目名称
  - 所属路径标签
  - 当前阶段（Phase 0-8）
  - **阶段进展指示器**（当前阶段 + 进度）
  - 状态指示器（正常/警告/阻塞）
- 支持按名称搜索项目
- 支持按状态筛选（全部/进行中/已完成/阻塞）
- 支持按路径筛选
- 支持按阶段筛选

### F2 项目详情视图

**描述**：点击项目卡片进入详情页，展示完整的项目信息。

**功能要求**：
- 显示项目基本信息（名称、描述、创建时间、当前阶段、所属路径）
- 显示项目所属 Agent 及各自的工作状态
- 显示该项目的任务列表（来自 `tasks/` 目录）
- 显示测试结果汇总
- 支持返回到项目列表

### F3 任务状态追踪

**描述**：展示各项目的任务完成情况。

**功能要求**：
- 读取 `<project>/.project/tasks/` 下的所有任务文件
- 任务状态包括：`waiting` | `in_progress` | `done` | `cancelled` | `paused`
- 任务类型包括：`dev` | `art` | `test` | `other`
- 计算并展示任务完成率
- 支持按任务类型筛选（**修复**：筛选后正确显示结果，不再空列表）
- 支持按状态筛选
- 支持按负责人筛选
- 任务卡片显示：标题、状态、负责人、截止时间

### F4 测试结果展示

**描述**：展示项目的测试执行结果。

**功能要求**：
- 读取 `<project>/test-results/` 或 `<project>/test/` 目录
- 统计测试总数、通过数、失败数、跳过数
- 显示测试通过率
- 展示失败的测试用例列表（如果有）
- 支持显示测试覆盖率（如果有）

### F5 Agent 工作状态

**描述**：展示各 Agent 在项目中的工作状态。

**功能要求**：
- 从 `status.json` 中读取 `waiting_for` 字段
- 显示当前等待哪个 Agent 处理
- 显示每个 Agent 的任务分布（待处理/进行中/已完成）
- 支持显示 Agent 活跃状态

### F6 数据可视化图表

**描述**：通过图表直观展示项目数据。

**功能要求**：
- **项目进展图表**：柱状图或饼图展示各项目完成度对比
- **任务分布图表**：展示不同状态任务的占比
- **测试趋势图表**：展示测试通过率趋势（如果有历史数据）
- **阶段分布图表**：展示各项目当前所处阶段的分布
- **路径分布图表**：展示各项目路径下的项目数量分布

### F7 自动刷新

**描述**：保持数据实时更新。

**功能要求**：
- 支持手动刷新按钮
- 支持设置自动刷新间隔（30秒/1分钟/5分钟/关闭）
- 刷新时显示加载动画

### F8 深色/浅色模式切换

**描述**：支持用户切换界面主题模式。

**功能要求**：
- 提供主题切换按钮（位于 Header 或设置菜单中）
- 支持两种模式：
  - **浅色模式（Light Mode）**：默认模式，适合明亮环境
  - **深色模式（Dark Mode）**：适合暗光环境，减少眼睛疲劳
- 切换时页面所有元素（背景、文字、卡片、图表等）整体切换主题
- 主题偏好应保存到本地存储，下次访问时自动应用
- 支持系统偏好自动切换（可选）：根据操作系统设置自动选择主题
- 切换动画流畅，过渡时间 200-300ms

### F9 多路径管理

**描述**：支持用户添加、编辑、删除项目监控路径。

**功能要求**：
- 提供路径管理界面（设置页面或独立管理面板）
- 支持添加新路径：**优先使用系统目录选择对话框**（而非手动输入路径）
- 支持编辑现有路径：修改路径别名/标签
- 支持删除路径：删除后该路径下的项目不再显示
- 支持路径排序：调整路径在列表中的显示顺序
- 路径有效性检查：定期检查路径是否存在，无效时提示用户
- 每个路径可设置：
  - 路径地址（必填）
  - 显示别名/标签（选填，默认使用路径名）
  - 颜色标签（选填，用于在项目卡片上标识来源路径）
- 支持导入/导出路径配置（JSON 格式）

### F10 Agent 任务视图

**描述**：展示各 Agent 在项目中的任务列表和状态。

**功能要求**：
- 按 Agent 分组显示任务列表
- 每个 Agent 显示：
  - Agent 名称和图标
  - 当前等待处理的任务（waiting）
  - 进行中的任务（in_progress）
  - 已完成的任务（done）
- 支持筛选特定 Agent 的任务
- 支持显示任务的截止时间和负责人
- 支持按状态筛选（waiting/in_progress/done）

### F11 项目阶段进展视图

**描述**：展示项目在 Software Workflow 中的当前阶段和进展。

**功能要求**：
- 根据 Software.md 定义，项目有 9 个阶段（Phase 0-8）：
  - Phase 0: 初始化
  - Phase 1: 需求分析
  - Phase 2: 设计
  - Phase 3: 开发
  - Phase 4: 测试
  - Phase 5: 部署
  - Phase 6: 维护
  - Phase 7: 完成
  - Phase 8: 已发布
- **阶段进度条**：可视化展示项目当前阶段 + 阶段内进度
- **阶段时间线**：展示项目经过的各阶段及耗时
- **阶段详情**：显示当前阶段的关键信息（如当前等待的 action）
- 支持显示 phase 和 waiting_for 信息

### F12 测试详情视图

**描述**：展示项目的测试详情和评审意见。

**功能要求**：
- **显示测试结果汇总**：总数、通过数、失败数、通过率（来自 `test-results/` 目录）
- **显示 Bug 统计**：
  - 总 Bug 数
  - 已修复 Bug 数
  - 未修复 Bug 数
  - Bug 修复进度百分比
- **显示评审意见汇总**（来自 `reviews/` 目录）
- **显示各版本测试结果**（如果有历史数据）
- 支持展示测试覆盖率趋势
- 支持按版本筛选测试结果

> ⚠️ **重要**：测试结果和 Bug 数据必须显示在 Dashboard 或项目详情页中，确保 Boss 能看到。

## 2.3 数据结构假设

### status.json 格式（期望）

```json
{
  "project_name": "char-counter",
  "phase": 3,
  "waiting_for": {
    "agent": "Designer",
    "action": "revise_requirements"
  },
  "since": "2026-03-18T10:00:00Z",
  "tasks_summary": {
    "total": 10,
    "done": 6,
    "waiting": 2,
    "in_progress": 2
  }
}
```

### Task 文件格式（期望）

```json
{
  "id": "TASK-001",
  "title": "实现字符计数功能",
  "type": "dev",
  "status": "in_progress",
  "assignee": "Engineer",
  "created_at": "2026-03-18T10:00:00Z",
  "due": "2026-03-20T18:00:00Z"
}
```

### 路径配置格式（期望）

```json
{
  "paths": [
    {
      "id": "path-001",
      "path": "C:\\Users\\Pheobe\\Projects",
      "alias": "主项目目录",
      "color": "#3B82F6",
      "enabled": true,
      "order": 0
    },
    {
      "id": "path-002",
      "path": "D:\\Work\\SideProjects",
      "alias": "副项目目录",
      "color": "#10B981",
      "enabled": true,
      "order": 1
    }
  ]
}
```

### 评审意见格式（期望）

```json
{
  "reviews": [
    {
      "round": "v_1",
      "role": "Designer",
      "conclusion": "APPROVE",
      "date": "2026-03-20",
      "comments": "..."
    }
  ]
}
```

### Bug 格式（期望）

```json
{
  "bugs": [
    {
      "id": "BUG-001",
      "title": "字符计数不准确",
      "severity": "high",
      "status": "fixed",
      "created_at": "2026-03-18",
      "fixed_at": "2026-03-19"
    }
  ]
}
```

## 2.4 边界情况处理

| 场景 | 处理方式 |
|:-----|:---------|
| 项目无 `.project/status.json` | 显示为"未初始化"状态，显示引导创建 |
| 项目无 `tasks/` 目录 | 显示"暂无任务" |
| 项目无测试目录 | 显示"暂无测试数据" |
| 筛选结果为空 | 显示"无匹配结果"提示，而非空列表 |
| 配置路径不存在 | 显示警告图标，点击可编辑或删除 |
| 无配置任何路径 | 显示引导页面，提示添加第一个路径 |
| 数据源文件格式错误 | 静默跳过，不显示错误弹窗；记录到日志 |
| 加载失败 | 显示内联提示，不显示阻断式弹窗 |

## 2.6 GitHub 版本管理（项目源码）

> ⚠️ **重要澄清**：GitHub 管理的是**项目管理系统本身的源码**，不是 Dashboard 中集成 GitHub 功能。

**要求**：
- 项目源码（整个仓库）提交到 GitHub private 仓库
- 打版本 tag：v0.1.0、v0.2.0 ...
- 阶段性进展后及时 commit + push
- `.project/` 目录纳入版本控制

**不在 Dashboard 中实现 GitHub 集成功能**（该功能为后续迭代）。
