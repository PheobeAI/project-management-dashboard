# 01 - 产品概述

## 1.1 产品定位

**项目管理平台** 是一个集中式的 Web 仪表盘，用于可视化展示多个自定义路径下各个软件项目的开发进展、任务完成度、测试状态和 Agent 工作情况。

## 1.2 核心目标

1. **统一视图**：在一个页面中概览所有项目的健康状态
2. **多路径管理**：支持添加多个项目目录路径，统一展示
3. **实时数据**：自动读取各项目的 `status.json`、`tasks/`、`test-results/` 作为数据源
4. **可视化呈现**：通过图表、进度条、状态卡片等形式直观展示数据
5. **现代化体验**：流畅的动画、精致的 UI，让项目管理成为一种享受
6. **深色/浅色模式**：支持深色和浅色主题切换，满足不同用户偏好
7. **阶段进展追踪**：清晰展示项目在 Software Workflow 中的当前阶段
8. **Agent 任务管理**：展示各 Agent 的任务列表和状态
9. **Bug 追踪**：展示 Bug 统计、修复进度、趋势
10. **版本历史**：展示项目版本迭代和变更记录

## 1.3 用户价值

| 用户 | 价值 |
|:-----|:-----|
| Boss | 快速了解所有项目的整体进展，识别风险项目 |
| PM | 集中管理项目状态，跟踪 Agent 工作，协调团队 |
| Engineer | 了解项目测试状态和待办任务 |
| Art | 了解设计任务进度 |
| QA | 跟踪 Bug 修复进度，查看评审意见 |

## 1.4 假设与约束

- **数据源假设**：每个项目遵循标准目录结构（`.project/status.json`、`tasks/`、`test-results/`）
- **多路径假设**：支持添加任意数量的目录路径到监控列表，每个路径下的子目录被视为独立项目
- **版本管理**：项目数据（包括 `.project/` 目录）使用 GitHub private 仓库管理
- **技术约束**：前端采用现代 Web 技术（React/Vue/HTML5），后端使用轻量级服务器
- **运行环境**：Windows 桌面环境

## 1.5 术语表

| 术语 | 定义 |
|:-----|:-----|
| 项目路径 (Project Path) | 用户配置的监控目录，支持多个路径 |
| 项目 (Project) | 项目路径下的每个子目录 |
| 任务 (Task) | 存储在 `<project>/.project/tasks/` 目录下的任务单元 |
| 阶段 (Phase) | 软件开发流程阶段（0-8），定义见 Software.md |
| Agent | AI 团队成员（Designer/PM/Engineer/Art/QA） |
| 深色模式 (Dark Mode) | UI 深色主题，减少眼睛疲劳 |
| 浅色模式 (Light Mode) | UI 浅色主题，默认显示模式 |
| Bug | 缺陷追踪，记录在 `bugs/` 目录 |
