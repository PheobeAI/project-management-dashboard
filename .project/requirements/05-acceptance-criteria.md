# 05 - 验收标准

## 5.1 核心验收条件

| 功能 | 验收条件 | 测试方法 |
|:-----|:---------|:---------|
| 项目列表 | 页面加载后展示所有已配置路径下的项目（排除隐藏目录） | 目视检查 |
| 项目卡片 | 每个卡片显示：名称、路径标签、进度百分比、状态指示器 | 目视检查 |
| 进度计算 | 进度百分比 = 已完成任务数 / 总任务数 × 100% | 对比数据源验证 |
| 状态指示器 | 正常（绿色）/ 警告（黄色）/ 阻塞（红色）根据规则显示 | 模拟不同状态验证 |
| 项目搜索 | 输入关键词可过滤项目列表 | 输入关键词测试 |
| 项目详情 | 点击卡片可进入详情页，显示完整信息 | 点击测试 |
| 任务列表 | 展示所有任务及其状态、负责人 | 对比 tasks/ 目录验证 |
| 测试结果 | 展示测试总数、通过数、失败数、通过率 | 对比测试目录验证 |
| 图表展示 | 页面包含至少一种数据可视化图表 | 目视检查 |
| 自动刷新 | 开启后数据按设定间隔更新 | 设置间隔观察更新 |

## 5.2 深色模式验收条件

| 验收项 | 验收条件 | 测试方法 |
|:-------|:---------|:---------|
| 切换按钮 | Header 或设置中存在主题切换按钮 | 目视检查 |
| 切换功能 | 点击按钮可切换深色/浅色模式 | 点击测试 |
| 背景变化 | 深色模式下页面背景变为深色（推荐 #0F172A 或 #1E293B） | 目视检查 |
| 文字变化 | 深色模式下文字变为浅色（推荐 #F8FAFC 或 #E2E8F0） | 目视检查 |
| 卡片变化 | 深色模式下卡片背景变为深色，与页面背景有区分 | 目视检查 |
| 图表适配 | 深色模式下图表颜色自动适配（使用亮色系图表） | 切换主题检查 |
| 状态保持 | 刷新页面后主题偏好保持不变 | 切换后刷新验证 |
| 过渡动画 | 主题切换有过渡动画，无闪烁 | 目视检查 |
| 系统适配 | 支持跟随系统深色模式偏好（可选） | 系统设置测试 |

## 5.9 状态指示器规则

| 状态 | 条件 |
|:-----|:-----|
| 🟢 正常 | 任务完成率 ≥ 80% 或无阻塞 |
| 🟡 警告 | 任务完成率 50%-79% |
| 🔴 阻塞 | `waiting_for` 字段存在且 `since` 超过 48 小时 |

## 5.10 视觉验收条件

- [ ] 页面布局整洁，元素对齐
- [ ] 卡片悬停有视觉反馈（颜色变化/阴影增强）
- [ ] 页面切换有过渡动画
- [ ] 数据加载显示骨架屏或加载动画
- [ ] 颜色方案专业、和谐（推荐蓝色系主色调）
- [ ] 字体清晰可读，层次分明
- [ ] **深色模式**：整体配色协调，对比度足够
- [ ] **深色模式**：卡片与背景有清晰区分
- [ ] **路径标签**：每个路径的颜色标签清晰可辨

---

## 5.11 性能验收条件

- [ ] 首屏加载时间 ≤ 3 秒
- [ ] 页面操作响应流畅，无明显卡顿
- [ ] 动画流畅，无掉帧
- [ ] 多路径（5+ 路径）下性能无明显下降

---

## 5.12 错误处理验收条件

- [ ] 数据源不存在时显示友好提示
- [ ] 网络错误时显示重试按钮
- [ ] 无效路径显示警告
- [ ] **加载失败不弹窗**：错误使用内联提示，不阻断用户操作
- [ ] **筛选为空不空列表**：筛选为空时显示"无匹配结果"提示
- [ ] 页面不白屏

---

## 5.13 验收流程

1. **环境准备**：确保 `$PROJECTS_DIR` 下有测试项目
2. **功能测试**：逐项验证上述验收条件
3. **深色模式测试**：分别验证深色和浅色模式
4. **多路径测试**：添加多个路径，验证项目显示正确
5. **Agent 视图测试**：验证 Agent 任务分组和筛选
6. **阶段进展测试**：验证 Phase 显示和时间线
7. **Bug 视图测试**：验证评审意见和 Bug 统计显示
8. **UI 修复测试**：验证错误不再弹窗、筛选正常、目录选择正常
9. **视觉检查**：检查 UI 设计和动画效果
10. **性能测试**：测量加载时间和响应速度
11. **最终确认**：所有条件通过后标记为验收完成
