## 页面总览

| 页面 | 入口 | 定位 | 核心价值 |
|:-----|:-----|:-----|:---------|
| Dashboard | 侧边栏首页 | 全局总览 | 一眼看全局：项目数、状态、阻塞、阶段、测试 |
| Projects | 侧边栏 | 项目列表+详情 | 深入每个项目的完整信息 |
| Agents | 侧边栏 | 团队工作负载 | 查看谁在做什么，任务分配是否均衡 |
| Tasks | 侧边栏 | 任务流转 | 跟踪每个任务的状态、优先级、截止时间 |
| Versions | 侧边栏 | 演进历史 | 了解项目的版本迭代和变更 |
| Documents | 侧边栏 | 文档中心 | 统一管理所有项目文档、评审记录（增强版） |
| Bugs | 侧边栏 | 质量追踪 | 追踪 Bug 总数、修复进度、趋势 |
| Settings | 侧边栏底部 | 系统配置 | 主题、路径、日志等系统设置 |
| Log | Settings 下 | 系统日志 | 监控异常：Agent 掉线、Task 停滞、加载失败 |

## 7.1 Dashboard（首页总览）

### 页面定位

**解决什么问题**：Boss 想要一眼看清所有项目的整体健康状态，不需要逐个点击。

**核心价值**：全局可视化，快速识别风险项目。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 统计概览 | 项目总数、进行中、已完成、阻塞数 |
| 项目卡片列表 | 可视化展示每个项目：名称、阶段、进度、状态 |
| 阶段进展时间线 | 展示所有项目的 Phase 分布 |
| 测试概览 | 汇总所有项目的 Bug 统计、测试通过率 |
| 快速筛选 | 按状态、路径、阶段筛选 |

### 用户流程

```
打开 Dashboard
  ↓
一眼看到统计概览（4个数字）
  ↓
扫视项目卡片列表，快速识别异常（红色警告）
  ↓
点击某个项目卡片 → 进入 Projects 详情
  ↓
或者直接点击侧边栏其他栏目
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 项目列表 | GET /api/projects |
| 统计数字 | GET /api/stats |
| 阶段分布 | GET /api/stats/by-phase |
| Bug 汇总 | GET /api/stats/bugs |

### 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  侧边栏                                                      │
├─────────────────────────────────────────────────────────────┤
│  Header: "Dashboard"                                        │
├─────────────────────────────────────────────────────────────┤
│  统计概览 (4 cards)                                         │
│  ┌────────┐ ┌────────┐ ┌────────┐ ┌────────┐           │
│  │总数: 12 │ │进行中:5 │ │已完成:4 │ │阻塞: 3  │           │
│  └────────┘ └────────┘ └────────┘ └────────┘           │
├─────────────────────────────────────────────────────────────┤
│  项目卡片列表 (Grid)                                         │
│  ┌─────────────────┐ ┌─────────────────┐                  │
│  │ char-counter    │ │ web-server     │                  │
│  │ Phase 3 ████░░ │ │ Phase 5 ██████ │                  │
│  │ 80%  🟢 正常   │ │ 100% ✅完成    │                  │
```

## 7.2 Projects（项目列表 + 详情）

### 页面定位

**解决什么问题**：深入了解每个项目的完整信息——状态、任务、Agent、测试、版本。

**核心价值**：项目级别的信息聚合，方便追踪单个项目的端到端状态。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 项目列表 | 展示所有项目的卡片，支持筛选 |
| 项目详情 | 点击进入完整详情页 |
| 状态概览 | Phase、进度、waiting_for、since |
| Agent 状态 | 项目团队成员及各自的任务 |
| 任务列表 | 该项目的所有任务 |
| 测试结果 | 该项目的测试汇总 |
| Bug 追踪 | 该项目的 Bug 列表 |
| 版本历史 | 该项目的版本/tag |

### 用户流程

```
从 Dashboard 点击项目卡片
  ↓
进入 Projects 详情页
  ↓
看到项目基本信息（Phase、状态、进度）
  ↓
查看 Agent 工作状态
  ↓
查看任务列表和进度
  ↓
查看测试结果和 Bug
  ↓
查看版本历史
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 项目列表 | GET /api/projects |
| 项目详情 | GET /api/projects/:id |
| Agent 状态 | GET /api/projects/:id/agents |
| 任务列表 | GET /api/projects/:id/tasks |
| 测试结果 | GET /api/projects/:id/test-results |

> 📌 **Boss 补充要求**：测试报告数据应从项目目录 `.project/test-results/` 下读取，包括 `TEST-*.md` 测试报告文件和 `bugs/` 子目录下的 Bug 报告。
| Bug 列表 | GET /api/projects/:id/bugs |
| 版本历史 | GET /api/projects/:id/versions |

### 布局结构

```
┌─────────────────────────────────────────────────────────────┐
│  Projects > char-counter                      [返回列表]    │
├─────────────────────────────────────────────────────────────┤
│  项目概览                                                    │

├─────────────────────────────────────────────────────────────┤
│  Agent 状态  │  任务列表  │  测试结果  │  Bugs  │ 版本  │
│  (Tab 切换)                                                    │
```

## 7.3 Agents（团队工作负载）

### 页面定位

**解决什么问题**：Boss 想看团队成员都在做什么，任务分配是否均衡，谁在超负荷工作。

**核心价值**：团队级别的视角，快速发现工作负载不均的问题。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| Agent 列表 | 展示所有 Agent（Designer/PM/Engineer/Art/QA） |
| 任务统计 | 每个 Agent 的 waiting/in_progress/done 任务数 |
| 任务列表 | 按 Agent 分组的任务详情 |
| 工作负载可视化 | 柱状图展示各 Agent 任务数对比 |
| 快速筛选 | 按 Agent、状态筛选 |

### 用户流程

```
打开 Agents 页面
  ↓
看到 5 个 Agent 的任务统计卡片
  ↓
发现某个 Agent 任务数明显偏高（如 Engineer 20个，其他 5个）
  ↓
点击该 Agent 卡片
  ↓
查看其具体任务列表
  ↓
决定是否重新分配任务
```

| 数据 | 来源 API |
|:-----|:---------|
| Agent 列表 | GET /api/agents |
| Agent 任务统计 | GET /api/agents/stats |
| Agent 任务详情 | GET /api/agents/:name/tasks |

## 7.4 Tasks（任务流转）

### 页面定位

**解决什么问题**：跟踪每个任务的状态、优先级、截止时间，确保没有任务被遗漏。

**核心价值**：任务级别的视角，关注任务的端到端流转。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 任务列表 | 展示所有任务的表格 |
| 状态筛选 | waiting/in_progress/done/cancelled/paused |
| 类型筛选 | dev/art/test/other |
| 负责人筛选 | 按 Agent 筛选 |
| 截止时间 | 显示任务截止时间，过期高亮 |
| 快速操作 | 查看任务详情、标记状态 |

### 用户流程

```
打开 Tasks 页面
  ↓
看到所有任务的列表（按状态分组）
  ↓
筛选：只看 waiting 的任务
  ↓
发现有几个 waiting 超过 3 天的任务
  ↓
点击查看详情
  ↓
或标记为 in_progress
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 任务列表 | GET /api/tasks |
| 任务统计 | GET /api/tasks/stats |
| 任务详情 | GET /api/tasks/:id |

## 7.5 Versions（版本历史）

### 页面定位

**解决什么问题**：了解项目的版本迭代历史，每次发布包含了哪些变更。

**核心价值**：项目演进的可视化，方便回溯和复盘。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 版本列表 | 展示所有版本（v0.1.0, v0.2.0...） |
| 版本详情 | 每个版本的创建时间、变更内容 |
| 变更记录 | 该版本关联的任务、Bug 修复 |
| 当前版本 | 标记最新版本 |
| 版本对比 | 可选：两版本之间的差异 |

### 用户流程

```
打开 Versions 页面
  ↓
看到版本列表（按时间倒序）
  ↓
点击某个版本（如 v0.3.0）
  ↓
查看该版本包含的变更：哪些任务完成、哪些 Bug 修复
  ↓
如果需要，回滚或对比
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 版本列表 | GET /api/versions |
| 版本详情 | GET /api/versions/:tag |
| 版本变更 | GET /api/versions/:tag/changes |


## 7.7 Bugs（B啦追踪）

### 页面定位

**解决什么问题**：追踪所有 Bug 的状态——总数、已修复、未修复、修复进度。

**核心价值**：质量追踪，让 Boss 知道项目的稳定性。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| Bug 统计概览 | 总数、已修复、未修复、修复率 |
| Bug 列表 | 所有 Bug 的列表 |
| 状态筛选 | open/in_progress/fixed |
| 严重程度 | high/medium/low |
| Bug 趋势 | 图表展示 Bug 数量变化趋势 |
| 负责人 | 分配给谁修复 |

### 用户流程

```
打开 Bugs 页面
  ↓
一眼看到 Bug 统计（总数 12，已修复 8，修复率 67%）
  ↓
看到 Bug 趋势图
  ↓
筛选：只看未修复的 high severity Bug
  ↓
点击某个 Bug 查看详情
  ↓
或指派给某个 Agent 修复
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| Bug 统计 | GET /api/bugs/stats |
| Bug 列表 | GET /api/bugs |
| Bug 详情 | GET /api/bugs/:id |
| Bug 趋势 | GET /api/bugs/trend |

## 7.9 Log（系统日志）

### 页面定位

**解决什么问题**：帮助 Boss 监控系统健康，快速发现异常情况——Agent 掉线、Task 长期停滞、加载失败等。

**核心价值**：运维视角，主动发现和预防问题。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 异常日志列表 | 按时间倒序展示所有异常和错误 |
| 日志分类 | Agent 异常 / Task 停滞 / 加载错误 / 系统错误 |
| 日志详情 | 点击查看完整错误堆栈和上下文 |
| 健康状态汇总 | 今日异常数、较昨日变化 |

### 日志类型定义

| 类型 | 触发条件 | 严重程度 |
|:-----|:---------|:---------|
| Agent 掉线 | Agent 超过 5 分钟无响应 | 🔴 高 |
| Task 停滞 | Task 状态为 waiting 超过 48 小时 | 🔴 高 |
| 加载失败 | 读取项目数据文件失败 | 🟡 中 |
| 数据格式错误 | JSON 解析失败 | 🟡 中 |
| 系统错误 | 后端服务异常 | 🔴 高 |

### 用户流程

```
打开 Settings > Log
  ↓
一眼看到今日异常汇总（异常数、较昨日变化）
  ↓
浏览日志列表，按类型筛选
  ↓
发现某个 Agent 掉线，点击查看详情
  ↓
决定是否手动重启 Agent 或通知相关人员
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 日志列表 | GET /api/logs |
| 日志详情 | GET /api/logs/:id |
| 健康状态 | GET /api/health |
| 日志分类统计 | GET /api/logs/stats |

> 📌 **实现说明**：系统需定期检查各 Agent 的活跃状态（心跳检测），发现异常时写入日志。当 Task 的 `since` 字段超过 48 小时，自动标记为停滞并写入日志。

## 7.10 Documents（增强版）

### 页面定位

**解决什么问题**：统一管理所有项目的文档——需求文档、设计文档、技术文档、评审记录。方便在任意时间快速查阅历史版本。

**核心价值**：文档的集中存储和版本追溯。

### 核心功能

| 功能 | 说明 |
|:-----|:-----|
| 文档树 | 按项目分类，展示目录结构 |
| 文档预览 | 在线查看 Markdown 内容 |
| 版本关联 | 文档与版本（tag）关联，可查看特定版本下的文档 |
| 搜索 | 按关键词搜索文档标题和内容 |
| 文档类型 | 需求文档 / 设计文档 / 技术文档 / 评审记录 |

### 文档分类

| 类型 | 路径模式 | 说明 |
|:-----|:---------|:-----|
| 需求文档 | `<project>/.project/requirements/*.md` | 需求阶段产物 |
| 设计文档 | `<project>/.project/design/*.md` | 设计阶段产物 |
| 技术文档 | `<project>/.project/tech-design/*.md` | 技术设计文档 |
| 评审记录 | `<project>/.project/reviews/**/DESIGNER.md` | 各轮评审意见 |
| 测试报告 | `<project>/.project/test-results/*.md` | 测试结果 |

### 用户流程

```
打开 Documents
  ↓
看到按项目分组的文档树
  ↓
展开某个项目，浏览其文档目录
  ↓
点击某份文档，预览内容
  ↓
或切换到"版本视图"，查看特定版本下的文档快照
```

### 数据来源

| 数据 | 来源 API |
|:-----|:---------|
| 文档树 | GET /api/documents/tree |
| 文档内容 | GET /api/documents/:path |
| 文档搜索 | GET /api/documents/search?q= |
| 版本列表 | GET /api/versions |

> 📌 **实现说明**：文档路径存储在数据库中，按项目分类。前端按目录树展示，支持展开/折叠。文档内容通过后端读取 Markdown 文件并渲染。
