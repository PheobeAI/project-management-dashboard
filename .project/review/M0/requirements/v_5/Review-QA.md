## 风险和建议

### ⚠️ 风险 1：Bugs 页面数据来源

**风险：** Bugs 板块需要统一的 Bug 数据源，但当前各项目的 Bug 报告是 Markdown 格式。

**建议：**
1. 实现 `test-results/bugs/index.json` 汇总所有 Bug
2. 或实现 Bug 数据解析服务，读取 Markdown 并转换为 JSON

### ⚠️ 风险 2：Documents 页面需要文件读取

**风险：** 文档预览功能需要读取实际文件内容。

**建议：**
1. 确认后端有文件读取权限
2. 验证 Markdown 渲染库已集成

### ⚠️ 风险 3：Versions 页面数据来源

**风险：** 版本历史数据从哪里来？需要与 Git 集成或手动维护。

**建议：**
1. 如果从 Git tag 读取，需要 Git 仓库路径配置
2. 如果手动维护，需要 API 支持创建版本
